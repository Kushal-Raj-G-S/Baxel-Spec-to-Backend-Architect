## The Elite NLP Tech Stack

### 1. Zero-Shot Entity Extraction: `GLiNER` (Instead of basic spaCy)
- **The Tech:** Generalist and Lightweight Named Entity Recognition paired with a Hardcoded Domain Implication Map.
- **The Pitch:** *"Instead of relying on legacy, hardcoded NLP rules, we use GLiNER—a state-of-the-art zero-shot extraction model. It dynamically identifies novel business entities (like 'Stripe Invoices' or 'IAM Roles') from unstructured text in sub-seconds. We then feed these entities through our Domain Implication Map (e.g., 'credit_card' automatically implies 'webhook_endpoint' and 'idempotency') to instantly provide deep architectural context without a massive LLM roundtrip."*

### 2. High-Dimensional Semantic Clustering: `BGE-M3` + `Faiss`
- **The Tech:** BAAI's `BGE-M3` (one of the best open-source embedding models) paired with Facebook's `Faiss` (Facebook AI Similarity Search).
- **The Pitch:** *"To prevent database fragmentation (e.g., creating separate tables for 'CEO' and 'Founder'), we embed all extracted actors into a high-dimensional vector space using BGE-M3. We then use Facebook's Faiss for hyper-fast vector similarity clustering, normalizing all synonymous roles into canonical entity definitions before generation."*

### 3. Intent Routing: `Semantic Router`
- **The Tech:** `semantic-router` (by Aurelio AI). 
- **The Pitch:** *"Before we even process the prompt, our Semantic Router categorizes the user's intent at lightning speed using vector embeddings. This allows us to instantly reject off-topic prompts (saving LLM costs) or route complex e-commerce requests to specialized architectural sub-agents. Ambiguous prompts trigger our **LOW_CONFIDENCE** route, ensuring we gracefully handle informal requests by warning the user while still attempting generation."*

### 4. Structured Enforcement: `Instructor` + `Pydantic`
- **The Tech:** The `instructor` Python library.
- **The Pitch:** *"We don't just 'hope' the LLM outputs the right JSON. We use strictly enforced Pydantic schemas compiled directly into the LLM's function-calling toolset, mathematically guaranteeing that the Next.js frontend receives a flawless architectural payload every single time."*

## Execution Strategy & Latency Optimization

To achieve SOTA latency without blocking, we execute the pipeline with the following optimizations:
1. `Semantic Router` checks if the prompt is valid and determines the archetype. Archetype classification results and their vector retrievals are cached (e.g., repeating similar requests skips the full DB query).
2. **Parallel Pipeline Branching:** We run the `GLiNER` entity extraction and the actor embedding / `Faiss` clustering processes in parallel using python's `asyncio`, as they are independent tasks.
3. The resolved entities and normalized actors are combined with the cached/retrieved blueprint constraints.
4. The compiled Intermediate Representation (IR) Pydantic schema is handed off to the generation layer.
